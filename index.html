History of the standard: beginnings

  • 1992: Tim Berners-Lee: Tags used in HTML
  • 1993: Hypertext Markup Language (HTML) Internet Draft
  • 1995: RFC 1866 - The ‘text/html’ Media Type

Source: (“HTML” 2017)

History of the standard: W3C Recommendations

  • January 1997: HTML 3.2 Reference Specification
  • December 1997: HTML 4.0 Specification: strict - frameset - transitional
  • 2000/2002: XHTML 1.0

Source: (“HTML” 2017)

History of the standard: WhatWG, W3C

  • 2004: Gründung der Web Hypertext Application Technology Working Group(WhatWG)
  • since 2006: Cooperation of WhatWG and W3C
  • July 2012: Degree of separation (WhatWG: HTML as living standard, W3C: Snapshots)
  • 2014: W3C-Specification HTML5

Sources: (“HTML” 2017), (“HTML5” 2017)

Design principles

Content first

  • Separation of content and presentation
  • Descriptive markup of content
  • Additional markup for machines

Progressive enhancement

  • Accessibility and future friendliness
  • Universal usability of the content
  • Device independence

Paragraphs

New Structural Elements in HTML5

div and span

  • div and span were (and are) used to structure HTML documents mainly for presentation purposes
  • They serve as a base for id and class-attributes
  • Their use has become obsolete by semantic HTML5 elements

Validation

What is validation?

XML validation is the process of checking a document written in XML (eXtensible Markup Language) to confirm that it is both well-formed and also “valid” in that it follows a defined structure. Source: (???)

A well-formed document follows the basic syntactic rules of XML, which are the same for all XML documents.

A valid document also respects the rules dictated by a particular DTD or XML schema. Source: (???)
